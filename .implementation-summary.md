## Architecture Highlights

### DRY (Don't Repeat Yourself)
- Single source of truth for mode logic in `gameMode.ts`
- Shared scoring function works for all modes
- Reuses existing Match infrastructure

### CLEAN (Clear Separation)
- Mode detection via type guards
- Negotiation logic isolated in separate module
- Clear API boundaries

### MODULAR (Composable)
- Bot strategy can be swapped independently
- Mode-specific logic doesn't pollute core game state
- Easy to add new modes in future

### BACKWARD COMPATIBLE
- `Match.mode` is optional (defaults to 'conversation')
- Existing conversation matches unaffected
- No breaking changes to existing APIs
